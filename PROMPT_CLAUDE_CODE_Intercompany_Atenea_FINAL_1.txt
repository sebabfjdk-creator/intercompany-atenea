## D. Parsers de ingesta — detalles técnicos críticos

### Colombia (Siesa)
- Balances: 2 filas de cabecera. Cuentas jerárquicas por longitud de código (1 → 11 → 1120 → 112005 → 11200501). Hoja `Balance_Enero` y `Balance_Febrero-Marzo`.
- Movimientos: columnas `Cuenta | Nombre | Tercero (NIT) | Nombre tercero | Referencia | F.creación | F.pago | Saldo anterior | Débito | Crédito | Saldo`. Hojas `Mvto_Enero` y `Mvto_Febrero-Marzo`.
- Números: punto decimal estándar. NIT en columna 3 (sin dígito de verificación generalmente).

### España (DELSOL) — el más complejo
- **Formato Libro Mayor impreso**: NO es tabla. Tiene metadatos de cabecera, bloques por cuenta.
- Detectar bloque: fila con patrón `^(\d{3}\.\d\.\d\.\d{3})\s+(.*)` = apertura de cuenta. Propagar ese código a cada movimiento del bloque.
- Descartar filas: `Anterior:`, `Total:`, `Total de cuenta:`, vacías, metadatos.
- Columnas de movimiento: `Fecha | Dia | Asto. | Ord | Concepto | Docum. | Debe | Haber | Saldo`.
- **Formato numérico español** — implementar `parse_es_number()`:
  ```python
  def parse_es_number(v) -> float:
      if v is None: return 0.0
      if isinstance(v, (int, float)): return float(v)
      s = str(v).strip()
      neg = s.endswith('-')
      s = s.rstrip('-').strip().replace('.', '').replace(',', '.')
      try: f = float(s)
      except: return 0.0
      return -f if neg else f
  # Tests: parse_es_number('4.500.000,00-') == -4500000.0 ✓
  #        parse_es_number('1.234.567,89 ') == 1234567.89 ✓
  ```

### Puente NIF↔NIT — función crítica
```python
import re

def normalizar_nif(nif: str) -> str:
    s = re.sub(r'[\s.]', '', str(nif).upper())
    s = re.sub(r'-[0-9A-Z]$', '', s)   # quitar dígito de verificación
    s = re.sub(r'[A-Z]$', '', s)        # quitar letra final
    return s.strip()

# Tests:
# normalizar_nif('901128135-5')     → '901128135'
# normalizar_nif('901207879S')      → '901207879'
# normalizar_nif('1.035.864.525-0') → '1035864525'
# normalizar_nif('70901974-N')      → '70901974'
# normalizar_nif('B12550877')       → 'B12550877' (NIF español, mantener)
```

**Lookup al ingerir**: cuenta ES 430/410 → `tercero_bridge.nif_normalizado` = NIT Colombia. Si NIF empieza con letra → entidad española (cruce por nombre como fallback).

### Etiquetado por concepto (movimientos ES sin código de tercero)
Para movimientos clase 6/7 donde el tercero aparece solo como texto en `Concepto`:
1. Limpiar concepto: quitar prefijo de fecha `MM/YYYY`, sufijos de factura `S. FRA: XXXX`, normalizar (mayúsculas, sin acentos, sin signos).
2. Buscar en `tercero_bridge.nombre_fiscal` con `token_set_ratio` (SequenceMatcher). Umbral: ≥ 0.65.
3. Asignar: score ≥ 0.85 → Alta; 0.70–0.84 → Media; 0.65–0.69 → Baja (marcar para revisión).
4. Guardar `movimiento.nit_co` y `movimiento.conf_tercero`.

**Nota futura:** el Libro Diario de España (asientos individuales con cuenta directa) permite atribución exacta sin necesidad de parsear el concepto. Diseñar el adapter con ese punto de extensión.

## F. Motor de conciliación

1. **Unidad**: grupo homologado × mes. `suma_neto_CO(mes)` vs `suma_neto_ES(mes)`.
2. **Saldo acumulado**: al cerrar el mes, calcular `running_balance` = suma de todos los meses hasta ese periodo.
3. **Diferencia**: `dif = neto_CO − neto_ES`. Guardar ambos lados + dif + % sobre base mayor.
4. **Estado**: `conciliado | excepción | en_revisión | aprobado | con_observación`.
5. **Causas de excepción**: redondeo, timing/periodificación, parafiscal_co (ICBF/SENA), diferencia_nif, error_imputación, sin_homologar, pendiente_libro_diario.
6. **Anotaciones por celda** en el resumen (igual que en la hoja Resumen del bosquejo).

## G. Funcionalidades del MVP

- Ingesta mensual: subir Excel → parser → BD → cierre automático del periodo.
- Gestión de homologación: cargar `Homologacion_de_cuentas_COMPLETA_Atenea.xlsx`, editable desde la app, reportar cuentas con movimiento aún sin homologar.
- Tablero **Comparativa**: por rubro, fila total CO + cuentas ES que lo componen + fila DIFERENCIAS por mes (réplica exacta del bosquejo).
- Tablero **Resumen**: por gran rubro, saldo acumulado, diferencias, anotaciones por celda, exportar a Excel/PDF.
- Vista **AR/AP por tercero**: para cada NIT, saldo Colombia (1305/22xx) vs saldo España (430/410), diferencia.
- Bandeja de **excepciones**: filtrar por rubro, mes, causa, estado. Workflow aprobación.
- **Detección de anomalías v1**: z-score sobre serie mensual por cuenta, cuentas sin homologar con movimiento, terceros con conf_tercero Baja.
- Módulo de **auditoría**: log inmutable, visible desde la app.
